# Raster-Code-Differences
The Raster Code Differences tool implements a pixel‑wise comparison of two temporally consecutive LULC rasters. Both input rasters must share the same spatial extent, cell size, projection, and classification nomenclature to ensure consistent alignment. For each pixel, the tool retrieves the numerical code of the earlier map (e.g., CLC 1990) and that of the later map (e.g., CLC 2006). These two values are then concatenated into a single unique identifier by applying the formula: (code₁ × 100,000) + code₂. This operation yields a transition raster where each pixel value encodes a specific land cover trajectory: the digits preceding the zero indicate the initial class, while those following the zero represent the final class. For example, the code 24300232 identifies a change from class 243 (agriculture with significant natural spaces) in 1990 to class 232 (sclerophyllous vegetation) in 2006. The same code appears for all pixels undergoing that identical transition, while unchanged pixels (e.g., 243>243) retain the simple code of the class (e.g., 243). This method allows for the systematic identification and quantification of dominant land cover transitions across the study area, providing the foundation for incorporating dynamic LULC information into subsequent susceptibility modelling
